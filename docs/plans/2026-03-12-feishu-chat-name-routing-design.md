# Feishu Group Name Routing Design

## Goal

让机器人在收到类似“把结果发到 `YY专用机器人群`”的指令时，能够根据群名定位飞书 `chat_id`，并把最终普通文本结果发送到目标群。

## Current Project Context

- 机器人当前只会把回复发送回触发消息所在的 `chat_id`。
- 现有运行时已经支持“模型输出内部指令，运行层执行”的模式，附件发送通过 `[[FEISHU_SEND_FILE:...]]` 和 `[[FEISHU_SEND_IMAGE:...]]` 实现。
- 机器人已经具备飞书 IM 读写权限，并且本地 SDK 暴露了 `im.v1.chat.search` 与 `im.v1.chat.list`。
- 实测 `im.v1.chat.search` 可以直接返回群 `name` 和 `chat_id`，例如：`YY专用机器人群 -> oc_38005ddbd091eb2bdbbbc7d382d746a3`。

## Scope

### In Scope

- 用户用自然语言要求“把最终结果发到某个群”
- 运行层根据群名解析目标 `chat_id`
- 只支持“普通文本结果跨群发送”
- 找不到群时给出明确失败提示
- 匹配到多个群时给出候选列表，不做猜测
- 目标群发送成功后，原会话只回“已完成，已发送到某群”

### Out of Scope

- 附件、图片、文档链接跨群发送
- 自动把进度文档分享到目标群
- 维护持久化群名缓存
- 模糊到拼音、同义词、别名的高级搜索
- 自动选择同名群中的一个结果

## Recommended Architecture

沿用现有“模型声明意图，运行层执行”的结构：

1. 模型在最终回复中用一条受控内部指令声明目标群名。
2. 运行层解析这条指令，调用飞书 `chat.search` 把群名解析为 `chat_id`。
3. 若目标群唯一命中，则把最终普通文本结果发到目标群。
4. 原始会话只收到完成提示，不再重复完整结果。

这样做的好处是：

- 模型负责理解自然语言里的“发到哪个群”
- 运行层负责安全执行和歧义处理
- 不让模型直接产出 `chat_id`，避免误发

## Directive Contract

新增一条仅供运行层识别的内部指令，例如：

`[[FEISHU_SEND_CHAT:YY专用机器人群]]`

约束：

- 指令单独占行
- 指令只表达“目标群名”，不包含 `chat_id`
- 指令之外的正文就是应该发送到目标群的普通文本结果
- 第一版只允许一个目标群指令；如果模型输出多个，运行层按错误处理并回退提示用户

## Matching Strategy

运行层使用 `client.im.v1.chat.search({ query })` 查询候选群，然后本地排序：

1. 优先精确同名匹配
2. 其次使用“名称包含查询词”的结果
3. 过滤无效或非正常状态的群
4. 以 `chat_id` 去重

结果处理：

- `0` 个候选：原会话回复 `未找到群：<groupName>`
- `1` 个候选：发送到目标群
- `>1` 个候选：原会话回复候选列表，请用户给更精确名字

## User-Facing Behavior

目标行为如下：

- 用户说“把结果发到 YY专用机器人群”
- 机器人正常完成任务后：
  - 目标群收到最终普通文本结果
  - 当前会话只收到：`已完成，已发送到 YY专用机器人群`

失败行为如下：

- 群名没搜到：`未找到群：YY专用机器人群`
- 搜到多个：列出候选群名，并提示用户说更精确的名字
- 目标群发送失败：原会话明确提示发送失败原因

## Reply Flow Changes

当前回复链路会把 `codex` 输出的正文直接发回当前 `chat_id`。新增能力后需要变成：

1. 解析附件指令和目标群指令
2. 先决定最终文本应该发往哪个 `chat_id`
3. 如果存在目标群：
   - 把最终文本发往目标群
   - 当前会话只发完成提示
4. 如果不存在目标群：
   - 保持当前行为

为了降低风险，第一版不改变附件发送链路；只对纯文本最终回复生效。

## Error Handling

- 指令存在但正文为空：视为错误，提示“目标群已识别，但没有可发送的文本结果”
- 多个目标群指令：视为错误，提示用户只指定一个群
- 目标群搜索接口异常：原会话提示飞书群搜索失败
- 目标群发送异常：原会话提示发送失败，不吞错

## Security and Safety Constraints

- 不允许模型直接输出 `chat_id`
- 多候选时绝不自动猜测
- 只在用户明确表达“发送到某群”时启用跨群发送
- 第一版只支持普通文本，避免把文件、图片、文档误发到错误群

## Testing Strategy

- 单元测试目标群指令解析
- 单元测试群搜索结果排序与歧义处理
- 单元测试“0 个 / 1 个 / 多个候选”的行为
- 集成测试回复链路在存在目标群时只向目标群发送正文，并向原会话发送完成提示
- 回归测试现有附件指令解析和普通回复路径不受影响
