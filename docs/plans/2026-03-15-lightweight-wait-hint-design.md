# Lightweight Wait Hint Design

## Goal

让“简单问答”类飞书消息不要默认创建进度文档，而是先显示飞书输入态；如果 8 秒内还没给出最终回复，再补一条简短等待提示，并在最终回复后自动撤回。

## Scope

- 只影响文本型、简单问答类消息
- 不改文件、图片、语音、跨群投递和长任务进度文档逻辑
- 保持复杂任务仍然沿用现有进度提示 / 进度文档

## Design

- 新增一个 lightweight helper，负责两件事：
  - 判断当前消息是否属于“简单问答”
  - 调度延迟等待提示，并支持在完成、失败、取消时自动撤回
- 主流程里如果命中“简单问答”：
  - 跳过 `progressReporter`
  - 保留 `typing_indicator`
  - 启动一个 `8s` 的延迟等待提示
- 如果任务在 8 秒内完成，等待提示不会发出去
- 如果等待提示已经发出，最终回复后自动 recall

## Heuristic

简单问答判断先走保守规则：

- 仅限 `message_type=text`
- 不含文件 / 图片 / 语音输入
- 文本长度较短，且看起来是问题句
- 如果包含明显任务型词汇，如“帮我生成 / 修改 / 收集 / 写脚本 / 发到某群”等，则不走轻提示流

## Expected Result

- 问答体验更轻，不会为了一个短问题创建文档
- 仍然保留“机器人活着”的反馈
- 长任务继续保持可观察的进度流
